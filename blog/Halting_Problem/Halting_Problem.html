The Halting Problem is to find an effective procedure that, given any Turing machine \( T_m \) will determine whether \( T_m \) will halt on input n or loop forever.

I will give a high-level proof which is an un-rigorous argument meant to illustrate the main idea of a proof with the details omitted.

Remark. All Turing machines can be seen as computing some function from positive integers to positive integers. When speaking of a specific Turing machine, I use an uppercase letter to denote the Turing machine and the same letter in lowercase to denote the function it computes.

Remark. Every Turing machine can be encoded into a natural number m. We denote the corresponding Turing machine \( T_m \).

The High-level Proof

Proof. Suppose that there exists a Turing machine H that computes the Halting function,

\[ h(m,n) = \begin{cases} 0, &\text{if } T_m(n) \text{ does not halt (loops)} \\ 1, &\text{if } T_m(n) \text{ does halt} \end{cases} \]

where m is some encoding of the algorithm for Turing machine \( T_m \) and \(n\) is an input.

Given H, we can build another Turing machine, L, which computes the function,

\[ l(n) = \begin{cases} halt, &\text{if } H(n,n) = 0 \\ loop, &\text{if } H(n,n) = 1 \end{cases} \]

In other words, if H says that Turing machine \( T_m \) halts on input m, then L does not halt on input m, and vice-versa. Let p be the encoding of the algorithm for L. Now, what does L do on input p?

From the definition of L,

\[ l(p) = \begin{cases} halt, &\text{if } H(p,p) = 0 \\ loop, &\text{if } H(p,p) = 1 \end{cases} \]

which means that

\[ l(p) = \begin{cases} halt, &\text{if L does not halt on input p} \\ loop, &\text{if L halts on input p} \end{cases} \]

So L halts on input p iff L does not halt on input p. This is a contradiction. Thus, our assumption of the existence of H must be false.

The Missing Details

To make this proof rigorous, one has to,

  1. Give a formal description of a Turing machine.
  2. Show how every Turing machine can be seen as representing a function from positive integers to positive integers.
  3. Show how every Turing machine can be encoded into a natural number from its description (algorithm).
  4. Show how to build L given H.

The details on how to do these can be found in the referenced textbook.
